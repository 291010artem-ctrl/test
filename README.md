# Telegram NFT/Username/Number Price Bot

Бот для проверки актуальной цены и истории продаж NFT-подарков,
юзернеймов и номеров Telegram на площадках mrkt, Portals, Tonnel,
Fragment и Getgems.

## Запуск

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # заполнить BOT_TOKEN и доступы к площадкам
python -m bot.main
```

## Сценарий использования

1. `/start` — выбрать, что искать: подарок, юзернейм или номер.
2. Для подарка — ввести номер, затем модель/название.
   Для юзернейма/номера — ввести сам юзернейм/номер.
3. Бот опрашивает все 5 площадок параллельно и присылает:
   - актуальную цену на каждой площадке (TON и ~USD по текущему курсу);
   - историю продаж (площадка, цена в TON, курс TON/USD на момент
     продажи и цена в USD на тот момент);
   - список площадок, где данные получить не удалось, и причину.

## Авторизация на площадках

| Площадка | Нужен Telegram-аккаунт? | Переменная |
|---|---|---|
| Getgems | Нет (публичный GraphQL) | — |
| Tonnel | Да, `web-initData` из LocalStorage | `TONNEL_INIT_DATA` |
| Portals | Да, токен `tma {...}`, обновляется раз в 1-7 дней | `PORTALS_AUTH_TOKEN` |
| mrkt | Да, bearer-токен через `/auth` | `MRKT_BEARER_TOKEN` |
| Fragment | Нет (HTML-скрейпинг, может ломаться) | — |

Токены для Tonnel/Portals/mrkt получаются через эмуляцию открытия их
Mini App из обычного Telegram-аккаунта — это отдельный технический
аккаунт, а не Bot API токен. Используйте выделенный номер, не основной
личный аккаунт, и храните session-файл как секрет.

Для автоматизации этого есть `scripts/get_tokens.py`:

```bash
pip install -r requirements-scripts.txt
# в .env заполнить TG_API_ID / TG_API_HASH (https://my.telegram.org)
python -m scripts.get_tokens   # первый запуск спросит телефон и код входа
```

Скрипт логинится под техническим аккаунтом, по очереди открывает Mini App
каждой площадки (тот же вызов `RequestWebView`, что делает клиент
Telegram), достаёт `initData`, для mrkt сразу меняет его на bearer-токен
через `/auth`, и печатает готовые строки для `.env`. Имена ботов/приложений
в словаре `MARKETS` — best-effort; если площадка не открывается, поправь
их по реальной ссылке `t.me/<bot>/<app>`. Токены Portals живут 1-7 дней —
запускай скрипт периодически (например, по cron).

## Структура

- `bot/main.py` — точка входа, polling.
- `bot/handlers.py` — FSM-диалог.
- `bot/aggregator.py` — параллельный опрос всех площадок + подмешивание
  исторического курса TON/USD.
- `bot/markets/*.py` — клиент под каждую площадку.
- `bot/ton_price.py` — текущий и исторический курс TON/USD (CoinGecko).
- `bot/formatting.py` — сборка текстового отчёта.

## Известные ограничения

- Эндпоинты Tonnel/Portals/mrkt/Getgems реверс-инжинерены сообществом
  и официально не документированы — могут измениться без
  предупреждения.
- Fragment не имеет API, используется скрейпинг публичных страниц
  аукционов — хрупко к изменению вёрстки.
- История продаж сейчас реализована полноценно только для Getgems;
  Tonnel/Portals/mrkt возвращают только текущую цену — добавить вызовы
  истории несложно, когда появятся реальные токены для тестирования.
