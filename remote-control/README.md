# 📱 Android Remote Panel

Веб-панель для удалённого управления Android-телефоном с ПК через **ADB**
(Android Debug Bridge). Открываешь панель в браузере на компьютере и:

- **видишь экран** телефона в реальном времени;
- **управляешь**: клик мышью = тап, перетаскивание = свайп, печать с клавиатуры;
- **навигация**: кнопки Назад / Домой / Недавние / Питание / Громкость;
- **дашборд**: модель, версия Android, разрешение, статус и температура батареи;
- **приложения**: список, запуск и остановка;
- **файлы**: просмотр `/sdcard`, загрузка на телефон и скачивание на ПК;
- **уведомления**: просмотр активных и отправка своего уведомления на телефон.

На сам телефон **ничего ставить не нужно** — только один раз включить
«Отладку по USB». Вся логика работает через штатный `adb` на твоём ПК.

---

## 1. Установи ADB (platform-tools)

- **Windows**: скачай [SDK Platform-Tools](https://developer.android.com/tools/releases/platform-tools),
  распакуй и добавь папку в `PATH` (или укажи путь через переменную `ADB_PATH`).
- **macOS**: `brew install android-platform-tools`
- **Linux (Debian/Ubuntu)**: `sudo apt install adb`

Проверь: `adb version` должно что-то вывести.

## 2. Включи отладку на телефоне

1. Настройки → «О телефоне» → 7 раз тапни по «Номер сборки» (включит режим разработчика).
2. Настройки → «Для разработчиков» → включи **«Отладка по USB»**.
3. Подключи телефон USB-кабелем и на телефоне подтверди «Разрешить отладку» с этого ПК.

Проверь: `adb devices` — должно показать твоё устройство со статусом `device`.

### (необязательно) Управление по Wi-Fi, без кабеля

```bash
adb tcpip 5555            # с подключённым по USB телефоном
adb connect 192.168.0.10:5555   # IP телефона (Настройки → Wi-Fi → сеть)
```

Кабель после этого можно отключить. В самой панели есть поле для `adb connect`.

## 3. Запусти панель

```bash
cd remote-control
npm install
npm start
```

Открой в браузере: **http://localhost:8787**

Нажми «▶ Экран» — начнётся трансляция. Готово.

## 🖥️ Отдельное приложение (.exe) — личный десктоп-вариант

Панель можно собрать в обычное десктопное приложение (Electron) — со своим окном
и иконкой, без запуска сервера вручную и без браузера. Просто двойной клик по ярлыку.

### Запустить как приложение (без сборки)
```bash
cd remote-control
npm install        # поставит и Electron (первый раз качается ~100–200 МБ)
npm run app        # откроется окно приложения
```

### Собрать установщик / портативный .exe (Windows)
```bash
npm run dist            # → dist/AndroidRemotePanel-1.0.0-x64.exe (установщик)
                        #   + AndroidRemotePanel-1.0.0-portable.exe (портативный)
npm run dist:portable   # только портативный .exe (не требует установки)
```
Готовые файлы появятся в папке `remote-control/dist/`.

> **Где собирать:** `.exe` собирается на **Windows** (electron-builder делает
> нативную сборку под текущую ОС). Собери на своей Windows-машине — это одна команда.
> Есть также `npm run dist:mac` и `npm run dist:linux`.

### Чтобы вообще ничего не устанавливать пользователю
По умолчанию приложению всё равно нужен `adb` (из системного PATH). Чтобы
`adb` ехал прямо внутри `.exe`, положи файлы platform-tools в папку
`remote-control/platform-tools/` **перед** сборкой:

- Windows: `adb.exe`, `AdbWinApi.dll`, `AdbWinUsbApi.dll`
  (из [SDK Platform-Tools](https://developer.android.com/tools/releases/platform-tools))

Тогда собранное приложение найдёт adb автоматически, и на ПК не придётся ставить
его отдельно. Порядок поиска adb: `ADB_PATH` → папка `platform-tools` рядом с
приложением → системный PATH.

### Иконка приложения (необязательно)
Положи `build/icon.ico` (Windows, 256×256) и `build/icon.png` (Linux/mac) —
electron-builder подхватит их автоматически. Без них будет иконка по умолчанию.

> На телефоне по-прежнему нужно один раз включить «Отладку по USB» — это
> требование Android, обойти его нельзя.

## Настройки (переменные окружения)

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `PORT`     | `8787`       | Порт веб-сервера |
| `HOST`     | `127.0.0.1`  | Адрес прослушивания. Оставь localhost для безопасности |
| `ADB_PATH` | `adb`        | Путь к бинарнику adb, если он не в `PATH` |

## Как это работает

```
Браузер (панель)  ──REST /api/*──►  Node-сервер  ──►  adb  ──►  Телефон
       ▲          ──WS /ws (стрим экрана + ввод)──►
```

- **Экран**: сервер снимает `adb exec-out screencap -p` (PNG) в цикле и шлёт кадры
  по WebSocket. Это даёт ~4–6 кадров/сек — достаточно для управления, но это не видео.
- **Ввод**: `adb shell input tap/swipe/text/keyevent`.
- **Дашборд**: `dumpsys battery`, `getprop`, `wm size`.
- **Приложения**: `pm list packages`, `monkey`, `am force-stop`.
- **Файлы**: `adb push` / `adb pull`, `ls`.

## ⚡ Нужно плавное видео (30–60 fps)?

Эта панель использует только `adb` (без сторонних бинарников), поэтому картинка
обновляется несколько раз в секунду. Для **плавного зеркала экрана** с тем же
управлением используй [**scrcpy**](https://github.com/Genymobile/scrcpy) —
эталонный open-source инструмент (`brew install scrcpy` / `apt install scrcpy`).
Панель удобна как единый пульт для действий, дашборда, файлов и уведомлений,
а scrcpy — когда нужна именно быстрая видеокартинка.

## ⚠️ Безопасность

- По умолчанию сервер слушает только `127.0.0.1` — доступ лишь с этого ПК.
  Не открывай его в интернет без аутентификации.
- Отладка по USB — мощный доступ к телефону. Разрешай её только доверенным ПК
  и выключай, когда не нужна.
