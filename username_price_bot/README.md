# 💎 NFT Username Price Bot

Telegram-бот, который оценивает **реальную цену NFT-юзернейма** Telegram и показывает:

- 📊 **оценку справедливой цены** (в TON и $) с уровнем достоверности и объяснением;
- 💰 **текущую цену / статус** на маркете (Fragment / GetGems);
- 📜 **историю продаж** (если была) — даты и цены;
- 👛 **на каких кошельках лежал** юзернейм (таймлайн владельцев).

Пользователь присылает юзернейм (`@durov`, `durov`, `t.me/durov`) — бот возвращает отчёт.

---

## Откуда берутся данные

| Источник | Что даёт | Надёжность |
|---|---|---|
| **TonAPI** (`tonapi.io`) | он-чейн истина: текущий владелец, история владельцев/кошельков, история продаж с ценами, курс TON/USD | 🟢 высокая — основной источник |
| **Fragment** (`fragment.com`) | текущая «витринная» цена и статус (аукцион/продажа) | 🟡 средняя — **нет официального API**, парсится HTML |
| **GetGems** (`api.getgems.io`) | альтернативный листинг на вторичном рынке | 🟡 best-effort — схема GraphQL неофициальная |

Юзернеймы Telegram — это NFT в коллекции **Telegram Usernames** в блокчейне TON, поэтому
он-чейн данные (владельцы и продажи) берутся напрямую из блокчейна через TonAPI — это
самый стабильный путь. Fragment и GetGems используются как дополнение для «рыночной» цены.

> ⚠️ **Важная честная оговорка.** У Fragment нет публичного API — страница юзернейма
> парсится эвристиками и может «поехать» при редизайне сайта. Поэтому каждый источник
> падает «мягко»: если Fragment/GetGems недоступны, бот всё равно отдаёт он-чейн данные
> из TonAPI. Оценка цены — приблизительная и **не является финансовой рекомендацией**.

## Быстрый старт

### 1. Токен бота
Создай бота через [@BotFather](https://t.me/BotFather) и получи `BOT_TOKEN`.

### 2. (Рекомендуется) Ключ TonAPI
Бесплатный ключ на [tonconsole.com](https://tonconsole.com) сильно поднимает лимиты
запросов. Без него публичный тариф `tonapi.io` тоже работает, но с ограничениями.

### 3. Установка и запуск
```bash
cd username_price_bot
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env        # впиши BOT_TOKEN и (опц.) TONAPI_KEY
python -m bot               # запуск (long polling)
```

После запуска напиши боту в Telegram любой юзернейм.

```
│       └── pricing.py     # эвристическая оценка цены
└── tests/                 # офлайн-тесты чистой логики
```

## Что проверить при первом «боевом» запуске

Сборка велась без доступа в интернет, поэтому формы ответов трёх внешних источников
стоит сверить с живыми данными (код к этому готов и переживёт расхождения):

1. **TonAPI DNS-резолв** (`GET /v2/dns/{name}.t.me`) — код достаёт адрес NFT из поля
   `item.address`. Если структура иная — поправь `TonApi.resolve_username_nft`.
2. **GetGems GraphQL** — запрос в `services/getgems.py` помечен как экспериментальный;
   при ошибке схемы источник просто отключается, на работу бота это не влияет.
3. **Fragment HTML** — селекторы цены (`.tm-value` и др.) и определение статуса в
   `services/fragment.py` могут потребовать подстройки под текущую вёрстку.

Эталонные формы данных задокументированы в `tests/test_tonapi_parse.py`.

## Настройки (`.env`)

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `BOT_TOKEN` | токен от @BotFather (**обязательно**) | — |
| `TONAPI_KEY` | ключ TonAPI (рекомендуется) | пусто |
| `GETGEMS_API_KEY` | ключ GetGems (опционально) | пусто |
| `USERNAMES_COLLECTION` | адрес коллекции Telegram Usernames | `EQCA14o1-…zwnPi` |
| `REQUEST_TIMEOUT` | таймаут запросов, сек | `15` |
| `CACHE_TTL` | кэш отчётов, сек | `120` |
| `LOG_LEVEL` | уровень логов | `INFO` |
