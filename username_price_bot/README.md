# 💎 NFT Username Price Bot

Бот **меню-ориентированный**: `/start` → главное меню → «💎 Оценка юзернеймов».
Сейчас оценка юзернеймов — единственная функция, но структура рассчитана на рост.

Пользователь присылает юзернейм (`@durov`, `durov`, ссылкой `t.me/durov` или просто
символами), а бот показывает **карточку с кнопками**:

- 💰 **Актуальная цена** — если продаётся: цена + кнопка «Купить»; если нет — честно «нигде не продаётся»;
- 📜 **История продаж** — все продажи (даты, цены) + кнопка на **TonViewer** с полной историей передач по кошелькам;
- 📊 **Примерная стоимость** — оценка сразу в **TON / USDT / ₽** с **погрешностью (±%)**, достоверностью и объяснением.

Везде есть кнопки навигации («⬅️ Назад», «⬅️ Меню»), чтобы не застрять.

Особые случаи:
- юзернеймы **короче 4 символов не оцениваются** (их нельзя создать в Telegram);
- если юзернейм **не является NFT** (свободен/не выпущен), бот это помечает и даёт
  оценку **по виду** (длина/паттерн), без истории продаж.

## Откуда берутся данные

| Источник | Что даёт | Надёжность |
|---|---|---|
| **TonAPI** (`tonapi.io`) | он-чейн истина: **факт и цена текущей продажи**, владелец, история продаж с ценами, курсы TON→USD/RUB | 🟢 высокая — основной источник |
| **Fragment** (`fragment.com`) | ссылка на страницу юзернейма (где купить) | 🟢 ссылка строится без парсинга |
| **GetGems** (`api.getgems.io`) | недавние продажи похожих юзернеймов (для оценки) + резервный листинг | 🟡 best-effort — схема GraphQL неофициальная |
| **TonViewer** (`tonviewer.com`) | ссылка на полную историю передач NFT по кошелькам | 🟢 ссылка по адресу NFT |

Юзернеймы Telegram — это NFT в коллекции **Telegram Usernames** в TON. Поэтому
**продаётся ли юзернейм прямо сейчас и за сколько** определяется по он-чейн-контракту
продажи (TonAPI), а **не** парсингом Fragment — это убирает ложные «активные аукционы»
для лотов, где аукцион давно закончился.

> ⚠️ **Честно:** оценка приблизительная и **не является финансовой рекомендацией**.
> Каждый источник падает «мягко»: если GetGems недоступен, бот всё равно отдаёт
> он-чейн данные из TonAPI.

## Как считается «примерная стоимость»

Главный принцип — **не вводить в заблуждение**. Логика (`bot/market.py` +
`bot/services/pricing.py`) разделена на два режима.

**Режим A — юзернейм продаётся прямо сейчас.** Рынок уже называет цену, поэтому
оценка привязана к ней: фиксированная цена ≈ потолок (берём чуть ниже), аукционная
ставка ≈ пол (может вырасти). Реальную цену покупки **никогда не раздуваем** до
табличной. Если она сильно отличается от похожих продаж — показываем флаг
(«возможно, завышена / выгодно»).

**Режим B — не выставлен.** Смешиваем реальные сигналы:
1. **Похожие недавние продажи** — медиана продаж юзернеймов той же длины **и** класса
   (слово / цифры / прочее). Это главный и самый свежий сигнал.
2. **Своя история, пересчитанная к сегодня** — прошлые продажи умножаются на
   коэффициент рынка по годам; чем продажа старше, тем меньше её вес.
3. **«Пол» категории** — типичная цена для длины/паттерна. Он может только
   **поднять** устаревшую/заниженную оценку и **никогда не тянет вниз** свежие данные.

Ключевые защиты от вранья:
- свежая реальная продажа или активный листинг **всегда важнее** синтетических таблиц;
- коэффициент роста рынка намеренно **консервативный** (~2× за 2022→2026, не 6×),
  потому что в TON цены росли не монотонно;
- при нехватке данных достоверность помечается **низкой**, диапазон расширяется, а
  заголовок меняется на «Грубая оценка» + явная пометка «не точная цена».

Пример (4-буквенный, продан давно за 50 TON, есть похожие ~500 TON):

```
📊 Оценка цены: ~401 TON (~$2,005)   (диапазон 281–561 TON) · достоверность: средняя
   • Похожие 4-симв. (буквы/символы) продаются ~500 TON (по 12 продажам)
   • Прошлая продажа 50 TON (~53 мес. назад) ≈ 104 TON сегодня с поправкой на рост рынка
```

### Насколько точна оценка (честно)

| Что есть по юзернейму | Достоверность | Комментарий |
|---|---|---|
| Активный листинг / ставка | 🟢 высокая | это и есть текущая цена рынка |
| Недавняя продажа (< 4 мес.) | 🟢 высокая | свежий факт по этому же NFT |
| Похожие свежие продажи | 🟡 средняя | зависит от того, насколько «похожи» |
| Только старая продажа (> 1 года) | 🔴 низкая | пересчёт по индексу — это прикидка |
| Совсем нет данных | 🔴 низкая | только типичная цена категории, широкий диапазон |

> Таблицы индекса по годам и типичных цен по категориям в `bot/market.py` — грубые
> значения по умолчанию; они **автоматически калибруются** по реальным недавним
> продажам коллекции (GetGems), когда те доступны, и их легко поправить вручную.

## Быстрый старт

### 1. Токен бота
Создай бота через [@BotFather](https://t.me/BotFather) и получи `BOT_TOKEN`.

### 2. (Рекомендуется) Ключ TonAPI
Бесплатный ключ на [tonconsole.com](https://tonconsole.com) сильно поднимает лимиты
запросов. Без него публичный тариф `tonapi.io` тоже работает, но с ограничениями.

### 3. Установка и запуск
```bash
cd username_price_bot
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env        # впиши BOT_TOKEN и (опц.) TONAPI_KEY
python -m bot               # запуск (long polling)
```

После запуска напиши боту в Telegram любой юзернейм.

## Что проверить при первом «боевом» запуске

Сборка велась без доступа в интернет, поэтому формы ответов TonAPI/GetGems стоит
сверить с живыми данными (код к этому готов и переживёт расхождения):

1. **TonAPI DNS-резолв** (`GET /v2/dns/{name}.t.me`) — код достаёт адрес NFT из поля
   `item.address`. Если структура иная — поправь `TonApi.resolve_username_nft`.
2. **TonAPI NFT и история** (`/v2/nfts/{addr}` и `/v2/nfts/{addr}/history`) — отсюда
   берутся текущая продажа (поле `sale`), владелец и история (`NftPurchase` /
   `NftItemTransfer`). Эталон формы — в `tests/test_tonapi_parse.py`.
3. **GetGems GraphQL** — запросы в `services/getgems.py` (похожие продажи + резервный
   листинг) экспериментальные; при ошибке схемы источник просто отключается, оценка
   опирается на встроенную модель категорий из `bot/market.py`.

Статус «продаётся / не продаётся» намеренно берётся **только из он-чейна (TonAPI)**, а
не из парсинга Fragment — это исключает ложные «активные аукционы».

## Настройки (`.env`)

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `BOT_TOKEN` | токен от @BotFather (**обязательно**) | — |
| `TONAPI_KEY` | ключ TonAPI (рекомендуется) | пусто |
| `GETGEMS_API_KEY` | ключ GetGems (опционально) | пусто |
| `USERNAMES_COLLECTION` | адрес коллекции Telegram Usernames | `EQCA14o1-…zwnPi` |
| `REQUEST_TIMEOUT` | таймаут запросов, сек | `15` |
| `CACHE_TTL` | кэш отчётов, сек | `120` |
| `LOG_LEVEL` | уровень логов | `INFO` |
